PART 1 — Influencer Experience (Full Markdown, Clean Version)

1. Overview
이 문서는 Collaboom 플랫폼의 인플루언서 측 사용자 경험(UX), 기능, 로직, 상태 조건, 예외 케이스를 명확하게 정의한다.
전체 흐름:
Landing Page → Google Login → Onboarding → Campaign Browsing → Campaign Detail → Apply → Dashboard → Shipping Tracking → Upload → Score Update

이 섹션은 모든 UI/기능/버튼 활성 조건과 Supabase 상태가 정확히 일치하도록 작성되었다.

2. User Requirements
미국 거주자만 (Admin이 수동 검증)


TikTok 팔로워 1,000명 이상 (Admin이 수동 검증 후 승인)


TikTok handle은 unique 해야 함


Google 계정 1개 + TikTok handle 1개 = Collaboom 계정 1개



3. Authentication (Google OAuth)
Influencer는 Google OAuth로만 로그인 가능.


로그인 성공 시:


Supabase auth.users에 user 생성


influencers 테이블에 해당 user의 influencer row 생성 (최초 1회)


TikTok handle 충돌 시 onboarding 단계에서 오류 메시지 발생.

14. Influencer Edge Cases
Onboarding incomplete
모든 캠페인 Apply disabled


TikTok handle 변경

 “Changing your TikTok handle may affect upload verification.”


Shipping delay / lost
Influencer가 issue report 제출


Admin이 penalty rollback 여부 판단


Apply redundancy
이미 신청한 경우, Apply 버튼 hidden


First Ghosting

Admin manual로 penalty를 4로 낮춰 접근 허용 가능



PART 2 — Admin Experience (Full Markdown, Clean Version)

1. Overview
이 섹션은 Collaboom 플랫폼의 Admin(TooMyung 내부 운영팀) 이 사용하는 모든 기능을 정의한다.
 Admin의 핵심 목표는 다음과 같다:
대규모 인플루언서(100~500명 이상)의 신청/승인을 효율적으로 처리


배송 CSV 매칭을 정확히 처리


업로드 검증을 명확하게 수행


Score / Penalty를 운영 정책에 따라 부여


각 인플루언서의 상세 기록을 남기고 관리


전체 캠페인의 현황을 빠르게 모니터링

6.1 Actions
Approve (Single)
Row status → approved


applications.approved_at timestamp 저장


Influencer에게 승인 이메일 발송


Approved count 증가 시 inventory와 비교


approved_count >= inventory 이면 status = full


Reject (Single)
Row status → rejected


Influencer에게 rejection email 발송


Bulk Approve
체크된 모든 pending row 승인


Score를 보면서 빠르게 승인 가능


inventory 초과될 경우 자동으로 full 상태로 전환


Bulk Reject
체크된 모든 pending row 거절

13. Admin Workflow Summary
1. Campaign 생성
Admin → Create Campaign → Publish
2. Influencer 지원
Influencer → Pending 상태로 등록
3. 승인
Admin Approve → email send → row = approved
4. 배송
Admin uploads CSV → shipped/update delivered
5. 업로드 검증
Admin manually marks uploaded or missed
6. Score/Penalty 반영
Rules에 따라 자동 또는 manual 적용
7. Notes
Admin adds operational detail for future reference
